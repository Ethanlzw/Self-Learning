2. 机器学习基础概念
监督、无监督、强化学习
监督学习：训练数据为 (x, y)，学习从输入到输出的映射。课程中回归、二分类、多分类、图像分类、目标检测都属于监督学习。
无监督学习：只有 x，没有标签 y。目标可能是聚类、异常检测、压缩、生成、去噪、发现数据结构。
强化学习：智能体在状态中选择动作并接收奖励，本课程只做概念介绍，不是主要考试实现重点。
回归与分类
回归：输出连续数值，例如电机速度、温度预测。常用损失：MSE。
二分类：输出两个类别或正类概率，例如健康/故障。常用 sigmoid + binary cross entropy。
多分类：输出多个类别概率，例如轴承故障类型或 MRI 四类。常用 logits + CrossEntropyLoss。
多输出任务：图像分割、深度估计、姿态估计等，课程中主要作为概念介绍。
参数与超参数
参数：训练过程中由优化器更新的权重和偏置。
超参数：人为设定并控制训练过程，例如学习率、batch size、epoch 数、隐藏层大小、dropout、数据增强、scheduler。
关键点：学习率通常是最先需要调好的超参数。过大可能发散，过小训练慢或停滞。
数据划分
Train：用于更新模型参数。
Validation：用于调超参数、早停、模型选择。
Test：只在最后使用，用于无偏最终评估。
时间序列不能随意 shuffle，因为未来信息会泄漏到过去。
医学图像和同一病人的多切片数据最好按 patient-level split，否则容易出现数据泄漏。MRI 项目中由于文件名没有可靠 patient/session ID，notebook 明确记录了这个限制。
3. W1：数字孪生、线性回归、MSE、梯度下降
核心概念
W1 用 DC motor 数字孪生说明：真实系统可以先用软件仿真产生数据，然后训练模型预测系统行为。

简化电机方程：

J * dω/dt = K * V - b * ω
dω/dt = (K * V - b * ω) / J
ω_new = ω_old + dω/dt * dt
含义：

J：转动惯量，越大越难改变速度。
K：电机常数，电压转为扭矩的比例。
b：粘性摩擦，速度越高阻力越大。
V：输入电压。
ω：输出角速度。
线性回归逻辑
最简单模型：

speed = w * voltage + b
目标是找到使预测误差最小的 w 和 b。损失函数使用 MSE：

MSE = mean((y_pred - y_true)^2)
MSE 的优点：

可微，适合梯度下降。
大误差被平方后惩罚更强。
适合连续数值回归。
局限：

对异常值敏感。
若任务是分类，MSE 通常不如交叉熵合适。
梯度下降
更新规则：

w := w - α * ∂J/∂w
b := b - α * ∂J/∂b
解释：

梯度指向损失上升最快方向。
减去梯度就是往损失下降方向移动。
α 是学习率，决定每一步走多远。
notebook 关键代码
class DCMotor:
    def __init__(self, J=0.1, b=0.5, K=2.0):
        self.J = J
        self.b = b
        self.K = K
        self.omega = 0.0

    def update(self, voltage, dt):
        torque_electrical = self.K * voltage
        torque_friction = self.b * self.omega
        torque_net = torque_electrical - torque_friction
        angular_acceleration = torque_net / self.J
        self.omega += angular_acceleration * dt
        return self.omega

4. W2：神经网络、反向传播、训练循环
神经网络基本块
一个神经元做：

z = w · x + b
a = activation(z)
多层网络把多个线性变换和非线性激活组合起来。没有激活函数时，多层线性网络仍然等价于一个线性模型。

常见激活：

ReLU：max(0, x)，隐藏层默认选择，快且缓解梯度消失。
Sigmoid：输出 (0, 1)，适合二分类概率或门控结构。
Tanh：输出 (-1, 1)，零中心，常用于 RNN/LSTM 内部候选状态。
Softmax：把多分类 logits 变成概率分布，总和为 1。
反向传播
反向传播的本质是链式法则：把最终损失对每一层参数的影响逐层向后传。

单样本简单例子：

y_hat = w * x
L = (y_hat - y)^2
∂L/∂w = 2 * (y_hat - y) * x
w_new = w - lr * ∂L/∂w
PyTorch 训练四件套
组件	作用
nn.Module	定义模型结构
loss/criterion	定义“错得多严重”
optimizer	用梯度更新参数
data/tensor	提供训练输入与标签
标准训练循环
for epoch in range(epochs):
    optimizer.zero_grad()
    predictions = model(X_train)
    loss = criterion(predictions, Y_train)
    loss.backward()
    optimizer.step()
重点：

optimizer.zero_grad() 必须在每次反传前调用，因为 PyTorch 默认累积梯度。
loss.backward() 计算所有参数梯度。
optimizer.step() 更新权重。
评估时使用 model.eval() 和 torch.no_grad()。

5. W3：二分类、Logistic Regression、FFT、故障检测指标
从线性回归到 Logistic Regression
线性回归输出任意实数，不适合直接表示故障概率。Logistic regression 用 sigmoid 把线性输出压到 (0, 1)：

z = w · x + b
p(y=1|x) = sigmoid(z) = 1 / (1 + exp(-z))
常用阈值：

if p >= 0.5 -> class 1
else -> class 0
Binary Cross Entropy
BCE = -[y log(p) + (1-y) log(1-p)]
优点：

对“自信但错误”的预测惩罚极大。
与概率解释匹配。
比 MSE 更适合二分类。
工业维护中的指标
混淆矩阵：

TP：真实故障，预测故障。
TN：真实健康，预测健康。
FP：真实健康，预测故障，误报。
FN：真实故障，预测健康，漏报。
指标：

Precision = TP / (TP + FP)：报警的质量。
Recall = TP / (TP + FN)：真实故障被抓住的比例。
F1 = 2PR / (P + R)：precision 与 recall 的平衡。
ROC-AUC：跨所有阈值的区分能力。
关键点：工业故障检测中 FN 往往比 FP 代价高得多，所以 recall 通常优先。

def generate_vibration_signal(is_faulty=False, duration=DURATION, fs=FS):
    t = np.linspace(0, duration, int(fs * duration), endpoint=False)
    sig = np.sin(2 * np.pi * F_SHAFT * t)
    sig += 0.3 * np.random.normal(size=len(t))
    if is_faulty:
        sig += 0.5 * np.sin(2 * np.pi * F_FAULT * t)
    return t, sig
def compute_fft_spectrum(vib, fs=FS):
    N = len(vib)
    window = scipy_signal.windows.hann(N)
    vib_windowed = vib * window
    fft_values = fft(vib_windowed)
    freqs = fftfreq(N, 1 / fs)
    pos_mask = freqs > 0
    return freqs[pos_mask], np.abs(fft_values[pos_mask]) / N

6. W4：多分类、Softmax、CrossEntropyLoss、轴承诊断
二分类到多分类
任务	输出层	损失
回归	1 个或多个线性输出	MSE/MAE
二分类	1 个 sigmoid 概率	BCE
多分类	K 个 logits	CrossEntropyLoss
Softmax：

softmax(z_i) = exp(z_i) / sum_j exp(z_j)
Cross entropy：

L = -sum_i y_i log(p_i)
PyTorch 关键点：

nn.CrossEntropyLoss() 输入应为 raw logits，不要手动加 softmax。
标签用整数类别索引，例如 0, 1, 2, 3，不需要手动 one-hot。
One-hot 的意义
类别编号本身没有大小含义。Normal=0、Ball=1 不代表 Ball 是 Normal 的两倍。One-hot 只是表达“正确类是哪一个”，避免错误的数值顺序含义。

Dropout
Dropout 在训练时随机关闭部分神经元，迫使网络不要依赖单一通路，从而降低过拟合。

model.train() -> dropout active
model.eval()  -> dropout disabled
FFT 与轴承故障
时域看“发生了什么”，频域看“是什么周期性机制产生了它”。

Normal：主要有轴转频和宽带噪声。
Outer race fault：BPFO 附近有稳定峰值。
Inner race fault：BPFI 及其边带，因为故障随内圈旋转进入/离开载荷区。
Ball fault：BSF 附近更复杂。
Nyquist 定理：

采样频率 >= 2 * 最高目标频率

7. W5：CNN 基础与工业缺陷图像分类
为什么图像不用普通 MLP
MLP 会把图像 flatten 成一维向量，丢失局部空间结构。同一裂纹出现在不同位置时，MLP 需要重新学习。CNN 用共享卷积核滑动检测局部模式，能更有效学习边缘、纹理和形状。

卷积层
卷积核是小窗口，例如 3x3。它在图像上滑动，对局部区域做加权求和，输出 feature map。

输出尺寸：

output = floor((n + 2p - f) / s) + 1
n：输入尺寸。
p：padding。
f：kernel/filter size。
s：stride。
Pooling
Max pooling：保留局部最大响应，常用于突出最明显特征并降低尺寸。
Average pooling：取局部平均，更平滑。
CNN 优势
参数共享：同一个滤波器可在图像各处检测同一模式。
局部连接：每个输出只看局部区域，参数量更少。
平移等变：物体轻微移动时，特征响应也随之移动。

8. W6：高级视觉、数据增强、学习率调度、迁移学习
经典 CNN 架构
LeNet-5：早期手写数字识别网络，卷积 + pooling + 全连接。
AlexNet：ImageNet 时代深度 CNN 代表。
VGG：大量 3x3 卷积堆叠，结构简单但参数多。
ResNet：残差连接解决深层网络训练退化问题。
Inception：多尺度卷积分支并行，常配合 1x1 卷积降维。
ResNet 残差连接
核心思想：

output = F(x) + x
网络只需学习残差 F(x)，如果某层不需要改变输入，可以接近学习 0，从而更容易训练深层网络。

数据增强
训练时随机改变图像，提高泛化能力。

常见增强：

几何：RandomResizedCrop、HorizontalFlip、VerticalFlip、Rotation、Affine。
光度：ColorJitter、GaussianBlur、RandomGrayscale。
注意：validation/test 不能使用随机增强，只能使用确定性 resize/normalize。

学习率调度
StepLR：每隔固定 epoch 让学习率乘以 gamma。
ReduceLROnPlateau：验证指标停滞时降低学习率。
CosineAnnealingLR：学习率按余弦曲线下降，常用于 fine-tuning。
迁移学习
使用在大规模数据（如 ImageNet）上预训练的模型，把通用视觉特征迁移到新任务。

两种模式：

Feature extraction：冻结 backbone，只训练最后分类头。
Fine-tuning：解冻部分或全部层，用较小学习率继续训练。

9. W7：无监督学习、K-means、异常检测、Autoencoder
无监督学习目标
聚类：发现数据分组。
异常检测：找离正常模式远的数据。
生成：学习数据分布并生成新样本。
压缩/去噪：学习低维表示。
K-means
算法：

选择聚类数 K。
随机初始化 K 个中心。
把每个点分配给最近中心。
把中心移动到所属点的均值。
重复直到收敛。
目标函数：

J = mean(||x_i - μ_c(i)||^2)
优点：

简单、快速、可解释。
缺点：

递归预测：

def recursive_forecast(model, initial_seq, steps):
    model.eval()
    curr_seq = initial_seq.clone().unsqueeze(0)
    predictions = []
    with torch.no_grad():
        for _ in range(steps):
            pred = model(curr_seq)
            predictions.append(pred.item())
            curr_seq = torch.cat((curr_seq[:, 1:, :], pred.view(1, 1, 1)), dim=1)
    return np.array(predictions)
关键风险：多步递归预测会累积误差，因为后续输入包含模型自己的预测。

11. W9：目标检测、YOLO、RT-DETR、mAP
图像分类 vs 目标检测
图像分类：整张图是什么类别。
目标检测：图中有哪些物体，各自位置在哪里。
目标检测输出：

class + bounding box
box = (x, y, w, h) 或 (x1, y1, x2, y2)
R-CNN 系列
R-CNN：先生成 region proposals，再逐个送入 CNN，准确但慢。
Fast R-CNN：整图先过 CNN，再对 RoI pooling，速度更快。
Faster R-CNN：用 Region Proposal Network 学习候选框，进一步端到端。
YOLO
YOLO 把检测看成一次前向传播的回归问题：

image -> grid/cells -> boxes + confidence + class scores
优点：

快，适合实时应用。
全局看图，不需要单独 region proposal。
缺点：

小目标、密集重叠目标可能更难。
早期 YOLO 依赖 NMS 和 anchor 设计，现代版本逐步改进。
DETR / RT-DETR
DETR：使用 transformer encoder-decoder，一次预测一组对象，去掉 anchors 和 NMS。
RT-DETR：更适合实时检测，使用高效 CNN + hybrid encoder，提高速度与收敛。
检测指标
IoU：预测框与真实框交并比。
Precision：预测出的目标中有多少是真的。
Recall：真实目标中有多少被检测到。
AP：某一类 precision-recall 曲线下面积。
mAP：所有类别 AP 的平均。
mAP@0.5：IoU 阈值 0.5，较宽松。
mAP@0.5:0.95：多个 IoU 阈值平均，更严格，是现代标准。

def make_transform(img_size, augment=False, num_channels=1, mean=None, std=None):
    preprocessing = [transforms.Grayscale(num_output_channels=num_channels)]
    if augment:
        preprocessing.extend([
            transforms.RandomResizedCrop(img_size, scale=(0.90, 1.00)),
            transforms.RandomRotation(8),
            transforms.RandomAffine(degrees=0, translate=(0.02, 0.02), scale=(0.95, 1.05)),
        ])
    else:
        preprocessing.append(transforms.Resize((img_size, img_size)))
    preprocessing.extend([
        transforms.ToTensor(),
        transforms.Normalize(mean=mean.tolist(), std=std.tolist()),
    ])
    return transforms.Compose(preprocessing)

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=NUM_CLASSES):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 16, kernel_size=3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Conv2d(16, 32, kernel_size=3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Conv2d(32, 64, kernel_size=3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64 * 8 * 8, 256),
            nn.ReLU(),
            nn.Linear(256, num_classes),
        )

    def forward(self, x):
        return self.classifier(self.features(x))
尺寸逻辑：

输入: 1 x 64 x 64
Conv+Pool 1 -> 16 x 32 x 32
Conv+Pool 2 -> 32 x 16 x 16
Conv+Pool 3 -> 64 x 8 x 8
Flatten -> 64*8*8 = 4096
Linear -> 256 -> 4 logits

Step 4：训练工具函数
项目把训练和验证封装为通用函数：

def run_epoch(model, loader, criterion, device, optimizer=None):
    is_train = optimizer is not None
    model.train(is_train)
    total_loss, correct, total = 0.0, 0, 0

    with torch.set_grad_enabled(is_train):
        for imgs, labels in loader:
            imgs, labels = imgs.to(device), labels.to(device)
            logits = model(imgs)
            loss = criterion(logits, labels)

            if is_train:
                optimizer.zero_grad()
                loss.backward()
                optimizer.step()

            total_loss += loss.item() * labels.size(0)
            correct += (logits.argmax(dim=1) == labels).sum().item()
            total += labels.size(0)

    return total_loss / total, correct / total
解释：

有 optimizer 时是训练；无 optimizer 时是验证。
torch.set_grad_enabled(is_train) 避免验证时计算梯度。
logits.argmax(dim=1) 得到预测类别。
CrossEntropyLoss 输入 logits 和整数标签。

MRI 口试高频问答
Q1：为什么项目先训练 SimpleCNN，再训练 ResNet-18？
A：SimpleCNN 是从零训练的 baseline，用来建立最低可用性能和可解释结构；ResNet-18 使用 ImageNet 预训练特征，通过迁移学习提升泛化。两者对比能证明高级方法是否带来真实提升。

Q2：为什么 ResNet-18 输入是 3 channel，而 MRI 是灰度？
A：ResNet-18 的预训练权重来自 ImageNet，第一层卷积期望 RGB 三通道输入。项目把灰度图转换/复制成三通道，并用 ImageNet mean/std，使输入分布更接近预训练设置。

Q3：为什么不能只看 accuracy？
A：accuracy 会掩盖类别级错误。医学或故障检测中某些漏检代价更高，应看 confusion matrix、per-class recall、precision 和 F1。

Q4：CrossEntropyLoss 为什么不加 softmax？
A：PyTorch 的 nn.CrossEntropyLoss 内部已经包含 log_softmax + negative log likelihood。模型输出 raw logits 即可，手动加 softmax 可能造成数值不稳定。

Q5：项目最重要的局限是什么？
A：不能从文件名可靠识别 patient ID，因此无法确认 patient-level split；如果同一病人的切片跨 split，测试集性能可能过高。此外，模型只是图像分类器，不是临床诊断系统。

Q6：augmentation 为什么要保守？
A：医学图像的结构方向和形态可能有诊断意义。过强增强可能改变病理特征。项目只用小角度 rotation、轻微 affine 和 crop，尽量模拟采集差异而不破坏医学结构。

15. 复习时最容易出错的点
把 test set 用来调参。
在 CrossEntropyLoss 前手动 softmax。
忘记 optimizer.zero_grad()。
评估时忘记 model.eval() 和 torch.no_grad()。
图像 normalization 与预训练模型不匹配。
对时间序列随机 shuffle 导致未来信息泄漏。
只看 accuracy，不看 recall/F1/confusion matrix。
医学项目不说明 patient-level split 风险。
把高置信度当成真实可靠性，忽略数据分布外样本。
用过强 augmentation 改变医学或工业缺陷语义。
16. 口试最后检查清单
你应该能用自己的话解释：

数据从哪里来，类别是什么，训练/验证/测试如何划分。
每个模型的输入和输出 shape。
为什么该任务用这个损失函数。
训练循环每一行的意义。
评估指标如何计算，代表什么错误类型。
结果中哪些类容易混淆，为什么。
项目方法相比 baseline 是否改进，以及改进原因。
该项目部署到真实工业/医学场景还缺哪些验证。
