# TAL220B + HX711 实验数据综合分析

## 1. 已有数据组成

当前数据文件夹中共有三组实验数据，每组分别包含 `force_N`、`raw_adc`、`valid`、`saturated` 四类信号。

| 数据组 | 主要文件 | 持续时间 | 判断 |
|---|---|---:|---|
| `noload` | `force_n_noload.mat`, `raw_adc_noload.mat`, `valid_noload.mat`, `saturated_noload.mat` | 约 45 min | 空载零点漂移数据 |
| `2514` | `force_2514.mat`, `raw_adc_2514.mat`, `valid_2514.mat`, `saturated_2514.mat` | 约 34.3 min | 约 24.26 N 恒载数据，可作为初步蠕变/稳定性数据 |
| `2995_0` | `force_2995_0.mat`, `raw_adc_2995_0.mat`, `valid_2995_0.mat`, `saturated_2995_0.mat` | 约 120 min | 加载后卸载数据，包含约 29.36 N 加载段和卸载回零段 |

数据质量判断：

- 所有数据中 `saturated = 0`，说明 HX711 未出现满量程饱和。
- `noload` 和 `2514` 的 `valid` 比例约为 10%，说明记录频率高于 HX711 有效更新频率，后处理时必须只使用 `valid == 1` 的点。
- `2995_0` 的 `valid` 比例约为 99.9%，是当前最完整、最有价值的数据。

## 2. 温度记录及其意义

手写温度记录为：

| 时间 | 温度 |
|---:|---:|
| 0 s | 24.8°C |
| 1820 s | 24.6°C |
| 3380 s | 24.6°C |
| 7200 s | 24.5°C |

整个 7200 s 过程中温度变化：

```text
Delta T = 24.5 - 24.8 = -0.3°C
```

该温度变化很小。根据 TAL220B 数据手册，零点温度系数和灵敏度温度系数通常按：

```text
±0.05 %FS / 10°C
```

估算。则 0.3°C 温度变化对应的最大温度影响约为：

```text
0.05 %FS / 10°C * 0.3°C = 0.0015 %FS
```

对于灵敏度温度影响，其相对变化约为：

```text
0.0015 % of reading
```

以 29.36 N 加载数据为例，灵敏度温度变化导致的力值影响约为：

```text
29.36 N * 0.0015 % = 0.00044 N
```

因此，在今天这组数据中，温度变化不足以解释 `2514` 中约 `0.1045 N` 的加载段变化，也不足以解释 `2995_0` 中加载段约 `0.0286 N` 的变化。当前观察到的主要变化更可能来自：

- 传感器 / 夹具机械蠕变；
- 加载后的机械稳定过程；
- 连接结构微小松动或应力释放；
- 零点恢复行为；
- 电气与采样系统的低频漂移。

## 3. 空载零点漂移数据如何使用

`noload` 数据应作为零点漂移和 deadband 估计依据。

注意：`force_N` 在该组数据中基本为 0，这是因为固件中 deadband 将小力值压制为 0。因此，不能用 `force_N` 判断零点漂移，必须使用 `raw_adc`。

当前分析结果：

| 指标 | 数值 |
|---|---:|
| 持续时间 | 约 45 min |
| 前 60 s raw 均值 | 128159.114 counts |
| 后 60 s raw 均值 | 127953.254 counts |
| raw 漂移 | -205.860 counts |
| 等效张力漂移 | -0.00458 N |
| 漂移斜率 | -0.000160 N/min |
| 后 60 s 噪声标准差 | 0.00444 N |

用途：

1. 用于估计零点漂移趋势；
2. 用于设置 deadband；
3. 用于判断是否需要地面状态下的慢速零点跟踪；
4. 用于飞行前零点检查阈值设计。

建议：

```text
deadband_N ≈ 3 * sigma_zero_N
```

当前估算：

```text
deadband_N ≈ 3 * 0.00444 N = 0.0133 N
```

如果考虑工程裕度，可取：

```text
deadband_N = 0.02 N
```

这与当前程序中使用的 `0.02 N` 是一致的。

参考依据：

- HX711 数据手册：必须区分有效读数和无效读数；
- TAL220B 数据手册：零点输出存在温度影响和零点误差；
- NIST Handbook 44：自动零点跟踪必须受限，不能在真实受载状态下自动归零。

## 4. `2514` 恒载数据

`2514` 数据可作为初步恒载稳定性或蠕变数据。

当前分析结果：

| 指标 | 数值 |
|---|---:|
| 加载段持续时间 | 约 34.29 min |
| 中位力值 | 24.257 N |
| 平均力值 | 24.263 N |
| 前 60 s 均值 | 24.266 N |
| 后 60 s 均值 | 24.370 N |
| 力值变化 | +0.1045 N |

该变化约为：

```text
0.1045 / 24.26 ≈ 0.43 % of measured load
```

该变化明显大于温度变化可解释的量级。因此不能简单归因于温度，应作为系统加载后稳定性 / 蠕变 / 夹具应力释放的初步证据。

局限：

- 持续时间不足 1 小时；
- 没有明确记录实际加载质量；
- 没有标记加载开始时刻；
- 不能单独区分蠕变、机械稳定过程和电气低频漂移。

用途：

1. 可作为报告中的初步恒载稳定性观察；
2. 可用于说明实际系统存在加载后慢变化；
3. 不建议作为最终蠕变定量结论的唯一依据。

参考依据：

- TAL220B 数据手册：传感器存在蠕变指标；
- OIML R 60：蠕变需要在恒定载荷和稳定条件下测试；
- JCGM 100 / GUM：该变化应进入不确定度或限制说明。

## 5. `2995_0` 加载-卸载数据

### 5.1 加载段

当前分析结果：

| 指标 | 数值 |
|---|---:|
| 加载段持续时间 | 约 63.16 min |
| 中位力值 | 29.357 N |
| 平均力值 | 29.357 N |
| 前 60 s 均值 | 29.377 N |
| 后 60 s 均值 | 29.348 N |
| 力值变化 | -0.0286 N |

该变化约为：

```text
0.0286 / 29.36 ≈ 0.098 % of measured load
```

该组数据可以作为当前实验中最可靠的恒载蠕变 / 稳定性分析数据。

温度在 0 到 7200 s 仅变化约 0.3°C，因此该加载段变化主要不应归因于温度。

### 5.2 卸载回零段

当前分析结果：

| 指标 | 数值 |
|---|---:|
| 回零段持续时间 | 约 55.00 min |
| 前 60 s 回零均值 | -0.0346 N |
| 后 60 s 回零均值 | -0.0432 N |
| 回零变化 | -0.00863 N |

说明：

- 卸载后读数接近零，但不是严格为零；
- 回零阶段仍存在缓慢变化；
- 不应假设持续加载后卸载会立即恢复到原始零点。

用途：

1. 用于飞行后零点检查策略；
2. 用于说明卸载后需要等待稳定或重新 tare；
3. 用于支持“零点跟踪只允许在地面松绳状态启用”的设计原则。

参考依据：

- TAL220B 数据手册：存在蠕变和零点恢复相关误差；
- OIML R 60：载荷传感器测试应考虑加载后回零；
- NIST Handbook 44：自动零点调整必须受限；
- JCGM 100 / GUM：回零误差应进入系统误差预算。

## 6. 如果只补一个多点静态标定，其它不补，是否可行

可以，但需要明确限制条件。

如果只补一个多点静态标定，则当前数据可以形成以下完整度：

| 项目 | 当前是否具备 | 说明 |
|---|---|---|
| 零点漂移 | 基本具备 | 有 45 min 空载 raw ADC，可估计零漂和 deadband |
| 恒载蠕变 | 基本具备 | `2995_0` 有约 63 min 加载段，`2514` 可作为辅助 |
| 回零恢复 | 基本具备 | `2995_0` 有约 55 min 卸载回零段 |
| 温度记录 | 基本具备 | 温度变化很小，可用于说明温度不是主要影响因素 |
| 多点标定 | 缺失 | 必须补 |
| 线性误差 | 缺失 | 依赖多点标定 |
| 迟滞 | 缺失 | 依赖加载 / 卸载多点标定 |
| 重复性 | 不充分 | 若不补重复实验，只能弱化处理 |
| 动态响应 | 缺失 | 若项目不做快速保护，可作为局限说明 |

因此，若只补一个实验，优先补：

```text
多点静态加载 / 卸载标定
```

推荐载荷点：

```text
0 %, 10 %, 20 %, 40 %, 60 %, 80 % of selected safe full-scale load
```

每个载荷点保持 5 min，取最后 60 s 均值。

补完该实验后，可以得到：

- 实际 `counts_per_N`；
- 零点偏置；
- 线性误差；
- 加载 / 卸载迟滞；
- 各载荷点稳定噪声；
- 当前 24 N 和 29 N 数据在标定曲线中的位置。

## 7. 如果其它数据不补，会有什么后果

### 不补完整 1 小时空载零点漂移

后果：

- 当前只有 45 min 空载数据；
- 可以估计零点漂移，但不能严格称为 1 小时零点漂移；
- 报告中应写为“45 min no-load zero drift observation”。

影响程度：中等。  
原因是当前空载漂移很小，且已有 raw ADC 数据。

### 不补温度多点实验

后果：

- 不能实测 `k_zero_T` 和 `k_span_T`；
- 不能建立系统级温度补偿模型；
- 只能引用 TAL220B 数据手册作为 B 类不确定度。

影响程度：较低到中等。  
原因是今天温度仅变化约 0.3°C，温度影响可用数据手册估算并说明其量级较小。

### 不补重复性实验

后果：

- 无法系统评估重复加载条件下的分散性；
- 重复性只能引用 TAL220B 数据手册或用单次标定稳定段噪声近似；
- 报告中的不确定度会更保守。

影响程度：中等。

### 不补动态响应实验

后果：

- 无法验证不同滤波参数对张力峰值和响应延迟的影响；
- 如果系统只用于离线实验分析，影响较小；
- 如果系统用于飞行张力保护，影响较大。

影响程度取决于项目目标。  
若课程项目重点是 drift 和 static measurement，可作为局限说明；若涉及实时安全保护，应补动态响应实验。

## 8. 现有数据在最终报告中的建议用法

最终报告可这样组织：

1. 使用多点静态标定补充 `counts_per_N`，作为全部力值换算依据；
2. 使用 `noload` 数据说明空载零点漂移很小，并给出 deadband 依据；
3. 使用 `2995_0` 加载段作为主要恒载蠕变 / 稳定性数据；
4. 使用 `2995_0` 卸载段作为回零恢复数据；
5. 使用 `2514` 数据作为第二个载荷水平下的辅助稳定性观察；
6. 使用温度记录说明实验过程中温度变化很小，因此当前漂移主要不是由温度变化造成；
7. 对未补充的温度多点实验、重复性实验和动态响应实验，在报告中作为限制项说明。

## 9. 最终判断

在只补一个多点静态标定的前提下，当前数据已经足够支撑一个以 load cell 漂移为主题的硕士课程项目，但需要清楚说明实验边界。

最合理的结论是：

```text
现有数据可用于分析零点漂移、恒载稳定性、蠕变趋势和卸载回零行为。
温度记录表明本次实验温度变化很小，温度不是主要漂移来源。
仍必须补充多点静态加载 / 卸载标定，以确定 counts_per_N、线性误差和迟滞。
如果不补其它实验，则温度补偿、重复性和动态响应只能作为限制项或基于数据手册的 B 类不确定度处理。
```
